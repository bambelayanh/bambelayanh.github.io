01

Contexte du projet

Dans le cadre de la SAE 1.01 (Science des Données 1), nous avons exploité une base de données relationnelle SQLite couvrant la période 2016–2025 et regroupant les classements Times Higher Education (THE) ainsi que des indicateurs socio-économiques nationaux : PIB par habitant, taux d'alphabétisation, migration nette, parité femmes/hommes, etc.

🎯 Objectif Explorer les liens entre développement socio-économique et performance académique à l'aide de requêtes SQL paramétrées, de traitements Python et d'un reporting structuré sur Excel — en s'interdisant toute autre bibliothèque que os, sqlite3 et openpyxl.

La base est organisée autour de quatre tables (régions, pays, universités, classements) reliées par des relations 1:N en cascade. Chaque membre du groupe — Yanh, Glaineta et Martin — a pris en charge un sous-ensemble de fonctions couvrant les deux axes « reporting » et « exploration ».

03

Méthodologie & implémentation

L'approche suit un pipeline en quatre étapes : connexion SQLite → construction de requêtes SQL paramétrées → structuration Python des résultats → export Excel avec graphiques via openpyxl.

1

Gestion de la connexion BDD

Deux fonctions partagées gèrent le cycle de vie de la connexion unique : ouvrir_connexion_bdd() configure row_factory = sqlite3.Row pour des résultats sous forme de dictionnaires, et fermer_connexion_bdd() assure une fermeture propre.

Bonnes pratiques appliquées :
  • Instance de connexion unique réutilisée dans tout le script
  • Vérification de l'existence du fichier .db avant connexion
  • Messages explicites en cas d'erreur ou de succès
2

Requêtes SQL paramétrées & dynamiques

Toutes les requêtes utilisent des marqueurs ? pour éviter les injections SQL. La fonction rechercher_universites() illustre la construction dynamique : les critères optionnels (pays, année, seuils de score) sont ajoutés conditionnellement via des listes de conditions concaténées avec " AND ".join(conditions).

Techniques SQL mobilisées :
  • JOIN multi-tables (jusqu'à 4 tables) avec clés primaires/étrangères
  • Agrégation AVG, COUNT, ROUND avec GROUP BY
  • CASE WHEN … END pour catégoriser PIB, mixité, migration
  • ORDER BY + LIMIT pour les classements Top/Bottom N
  • HAVING pour filtrer les pays avec ≥ 20 universités
3

Structuration Python des résultats

Selon la fonction, les résultats sont renvoyés sous forme de liste de tuples [(valeur, …), …], de dictionnaire plat {"clé": valeur} ou de dictionnaires imbriqués {"catégorie": {"moy_ens": …, "moy_rech": …}}.

Objets Python utilisés :
  • fetchone() → dict() pour les KPIs agrégés (ligne unique)
  • fetchall() + [tuple(ligne) for ligne in …] pour listes de tuples
  • Boucle for + dictionnaires imbriqués pour stats par catégorie
  • Pondération score × nb/(nb+20) pour top10 multi-années
4

Reporting Excel avec openpyxl

Chaque fonction possède une feuille dédiée dans resultats_analyse.xlsx. Les données sont insérées ligne par ligne avec append(), puis des graphiques (barres, camembert, courbes) sont générés automatiquement par les trois fonctions de visualisation fournies.

Feuilles générées :
  • KPIs – Vue Globale · Top5 Pays–Enseignement(2025) · Top5 Pays–Recherche(2025)
  • Top10 Pays–Ens(2020–2025) · Univ par Région · Top 20 Univ (Index)
  • Evolution-Univ · Internat vs Recherche · PIB Par Hab vs Perf
  • Top & Alphabétisation · Mixité vs Perf · Migration vs Perf · Recherche Univ

def rechercher_universites(conn, pays=None, annee=None,
                          min_enseig=None, min_rech=None, limite=50):
    """Filtre les universités selon des critères optionnels, triées par rang."""
    # Validation des types d'entrée
    if pays is not None and not isinstance(pays, str):
        raise ValueError("'pays' doit être une chaîne de caractères")
    if annee is not None and not isinstance(annee, (int, float)):
        raise ValueError("'annee' doit être un entier")

    # Requête SQL de base
    req = """
    SELECT c.rang, u.nom_univ, p.nom_pays,
           c.score_enseig, c.score_qualite_rech, c.score_the
    FROM classements c
    JOIN universites u ON c.id_univ = u.id_univ
    JOIN pays p ON u.id_pays = p.id_pays
    WHERE p.pib_par_hab >= 0
    """

    # Ajout dynamique des critères optionnels
    conditions = []
    parametres = []

    if pays is not None:
        conditions.append("p.nom_pays = ?")
        parametres.append(pays)
    if annee is not None:
        conditions.append("c.annee = ?")
        parametres.append(annee)
    if min_enseig is not None:
        conditions.append("c.score_enseig >= ?")
        parametres.append(min_enseig)
    if min_rech is not None:
        conditions.append("c.score_qualite_rech >= ?")
        parametres.append(min_rech)

    if conditions:
        req += " AND " + " AND ".join(conditions)  # pas de WHERE dupliqué !

    req += " ORDER BY c.rang LIMIT ?"
    parametres.append(limite)

    curseur = conn.cursor()
    curseur.execute(req, parametres)
    return [tuple(ligne) for ligne in curseur.fetchall()]
def stats_pib_vs_perf(connexion, annee=2025):
    """Catégorise les universités selon le PIB/hab du pays et calcule
    la moyenne de score_enseig, score_qualite_rech, score_the."""
    curseur = connexion.cursor()
    req = """
    SELECT
        ROUND(AVG(CASE WHEN p.pib_par_hab <= 1135
                       THEN c.score_enseig END), 4) AS moy_ens_faible,
        ROUND(AVG(CASE WHEN p.pib_par_hab BETWEEN 1136 AND 4495
                       THEN c.score_enseig END), 4) AS moy_ens_inter,
        ROUND(AVG(CASE WHEN p.pib_par_hab >= 4496
                       THEN c.score_enseig END), 4) AS moy_ens_eleve,

        ROUND(AVG(CASE WHEN p.pib_par_hab <= 1135
                       THEN c.score_qualite_rech END), 4) AS moy_rech_faible,
        ROUND(AVG(CASE WHEN p.pib_par_hab BETWEEN 1136 AND 4495
                       THEN c.score_qualite_rech END), 4) AS moy_rech_inter,
        ROUND(AVG(CASE WHEN p.pib_par_hab >= 4496
                       THEN c.score_qualite_rech END), 4) AS moy_rech_eleve,

        ROUND(AVG(CASE WHEN p.pib_par_hab <= 1135
                       THEN c.score_the END), 4) AS moy_the_faible,
        ROUND(AVG(CASE WHEN p.pib_par_hab BETWEEN 1136 AND 4495
                       THEN c.score_the END), 4) AS moy_the_inter,
        ROUND(AVG(CASE WHEN p.pib_par_hab >= 4496
                       THEN c.score_the END), 4) AS moy_the_eleve
    FROM classements AS c
    JOIN universites AS d ON c.id_univ = d.id_univ
    JOIN pays AS p ON d.id_pays = p.id_pays
    WHERE c.annee = ?
    """
    curseur.execute(req, (annee,))
    ligne = curseur.fetchone()  # fetchone() car une seule ligne de résultats

    return {
        "Faible":       {"moy_ens": ligne[0], "moy_rech": ligne[3], "moy_the": ligne[6]},
        "Intermédiaire": {"moy_ens": ligne[1], "moy_rech": ligne[4], "moy_the": ligne[7]},
        "Elevé":        {"moy_ens": ligne[2], "moy_rech": ligne[5], "moy_the": ligne[8]},
    }
def top10_pays_multi_annees(connexion, annee_deb=2020, annee_fin=2025,
                            critere="score_enseig"):
    """Top 10 des pays avec ≥ 20 universités, score pondéré sur la période."""
    pascur = connexion.cursor()

    # Moyenne + comptage par pays, filtrée sur la période
    req = f"""
    SELECT p.nom_pays, AVG(c.{critere}) AS moy, COUNT(DISTINCT u.id_univ) AS nb
    FROM classements c
    JOIN universites u ON c.id_univ = u.id_univ
    JOIN pays p ON u.id_pays = p.id_pays
    WHERE c.annee BETWEEN ? AND ?
    GROUP BY p.nom_pays
    HAVING nb >= 20
    """

    res = []
    # Pondération : score × nb / (nb + 20)  → évite qu'un pays à 1 univ top parassite
    for r in pascur.execute(req, (annee_deb, annee_fin)):
        score = r["moy"] * r["nb"] / (r["nb"] + 20)
        res.append((r["nom_pays"], round(score, 3), r["nb"]))

    return sorted(res, key=lambda x: x[1], reverse=True)[:10]

Résultats & enseignements

🏆

Leaders académiques stables

Les États-Unis, le Royaume-Uni et Singapour dominent tous les critères. Un écart marqué sépare les 3 premiers pays du reste du classement.

📈

Convergence des indicateurs

Pour Panthéon-Sorbonne (Paris 1), enseignement, recherche et score THE convergent vers l'intervalle [32–38], avec une nette amélioration en recherche depuis 2020.

🌍

Alphabétisation ↔ présence universitaire

68 % des universités du classement appartiennent à des pays avec un taux d'alphabétisation > 90 %, soulignant le lien étroit entre niveau éducatif national et rayonnement universitaire.

🌐

Internationalisation & excellence

Les universités accueillant plus de 30 % d'étudiants internationaux affichent un score moyen de recherche de 73,95, contre 44,68 pour celles sous les 10 %.

💰

PIB/hab & performances

Les pays à PIB élevé (≥ 4 496 $/hab) affichent des scores d'enseignement et de recherche nettement supérieurs, bien que des exceptions notables existent.

⚖️

Parité et performance

Les universités mixtes (40–60 % de femmes) présentent les meilleures performances moyennes en enseignement et en recherche — la parité est corrélée à l'excellence.

05

Défis rencontrés & solutions

  • 🔗 Jointures multi-tables et doublons de lignes

    Les jointures simultanées entre les 4 tables pouvaient multiplier les lignes et fausser les agrégats. Solution : calcul des KPIs via des sous-requêtes séparées par table, garantissant des valeurs exactes.

  • ⚡ Requête SQL dynamique avec critères optionnels

    Construire une requête dont le nombre de clauses WHERE varie selon les paramètres non-None. Solution : listes conditions et parametres remplies conditionnellement, puis concaténées avec " AND ".join(conditions).

  • 📊 CASE WHEN vs traitement Python

    La première version de stats_pib_vs_perf utilisait 9 listes Python (68 lignes). L'utilisation de CASE WHEN … END en SQL a réduit le code à 35 lignes, plus lisibles et plus performantes.

  • 📐 Pondération pour un classement équitable

    Un pays avec une seule université très performante pouvait dominer artificiellement le Top 10. La formule score × nb/(nb+20) et le filtre HAVING nb ≥ 20 ont résolu ce biais.
